Run extractor on IronPython 3 (work around pyRevit 6.x CPython bug)

"input string '3.12.3' was not in a correct format". The extractor is
stdlib-only and IronPython-safe (no f-strings, %-formatting), so drop the
'#! python3' shebang and run on the stable default IronPython 3 engine.
